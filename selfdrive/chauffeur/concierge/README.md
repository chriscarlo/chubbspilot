# Concierge Web Server

Concierge is a web-based management interface for openpilot that runs on port 5055.

## Status: ✅ Fully Operational (June 10, 2025)

The Concierge web interface is running with a complete terminal emulator implementation.

## Features

- Real-time system status monitoring
- Crash log viewer
- Service monitoring
- Map data log viewer
- **Terminal interface** - Full web-based terminal emulator with PTY support

## Access

When enabled, access the interface at:
- http://device-ip:5055 (from any device on the same network)
- http://localhost:5055 (from the device itself)

## Understanding the Status Display

### Health Score
The health score (e.g., 2/6) indicates how many of the following checks are passing:
1. **Process Running** - The web server process is active
2. **Port Open** - Port 5055 is accessible
3. **HTTP Responding** - The web server is responding to requests
4. **Dependencies OK** - All required Python packages are installed
5. **No Recent Errors** - No errors in the last few minutes
6. **Manager Running** - The process manager is active

### Common Issues and Solutions

#### "Process not running"
- Toggle the Concierge switch OFF then ON again
- Wait 10-15 seconds for the process to start

#### "HTTP Connection refused"
- The process is not running or still starting up
- Check if port 5055 is blocked by firewall
- Ensure no other service is using port 5055

#### "Dependencies Missing"
- The system will try to auto-install missing packages
- If on a development system, manually install:
  ```
  pip install fastapi uvicorn jinja2 pydantic
  ```

#### "Port 5055 already in use"
- Another process is using the port
- Find and stop the conflicting process:
  ```
  sudo netstat -tlnp | grep 5055
  ```

## Security Notice

The web interface is accessible to all devices on your network. Only enable it on trusted networks.

# Concierge Frontend Styling with Tailwind CSS v4

This document outlines how to manage and build the frontend CSS for The Concierge application using Tailwind CSS v4.x.

## Overview

The project uses Tailwind CSS v4.x, which employs a CSS-first configuration approach. Styles are defined primarily within `css/input.css` and then compiled into a static asset.

**Key Files:**

*   `css/input.css`: The main source file for Tailwind CSS. This is where you import Tailwind, define custom themes, and can add custom CSS or Tailwind directives.
*   `static/css/tailwind.min.css`: The compiled, minified CSS output that is served to the browser. **Do not edit this file directly.**
*   `package.json`: Contains the build script for compiling the CSS.
*   `templates/index.html` (and other HTML files): Where Tailwind utility classes are applied.

## Development Prerequisites

To work on the frontend styling and compile the CSS, you will need:

*   **Node.js and npm:** Required to install Tailwind CSS and run the build scripts. Install from [nodejs.org](https://nodejs.org/) or via your system's package manager.

## Generating CSS

To compile `css/input.css` into `static/css/tailwind.min.css`, run the following command from the `selfdrive/chauffeur/concierge/` directory:

```bash
npm run build:css
```

This command executes the script defined in `package.json`:
`"tailwindcss -i ./css/input.css -o ./static/css/tailwind.min.css --minify"`

**Important:** Ensure you have `tailwindcss` and `@tailwindcss/cli` (version 4.x) listed in your `devDependencies` in `package.json` and that they are installed via `npm install`.

## Configuration & How It Works (Tailwind v4 CSS-First)

Tailwind CSS v4.x shifts configuration from a `tailwind.config.js` file primarily into your main CSS input file (`css/input.css`).

1.  **Import Tailwind:**
    The `css/input.css` file must start with:
    ```css
    @import "tailwindcss";
    ```
    This replaces the old `@tailwind base; @tailwind components; @tailwind utilities;` directives from v3.

2.  **Content Scanning (`@source`):**
    Tailwind v4 attempts automatic content detection. To ensure your template files (HTML) and any JavaScript files manipulating classes are scanned, you can use the `@source` directive (paths are relative to `input.css`):
    ```css
    @source "../templates/**/*.html";
    @source "../static/js/**/*.js";
    ```

3.  **Custom Theme (`@theme`):**
    Define your custom theme (colors, fonts, breakpoints, etc.) using CSS custom properties within an `@theme` block. Tailwind v4 will automatically make these available as utility classes.
    ```css
    @theme {
      /* Custom Colors */
      --color-1: #0d1117;       /* Generates .bg-1, .text-1, .border-1 etc. */
      --color-primary: #c9d1d9; /* Generates .text-primary, .bg-primary etc. */
      /* ...add all your custom color tokens here ... */

      /* Custom Fonts */
      --font-sans: 'Helvetica', 'Arial', 'sans-serif'; /* Generates .font-sans */
      /* ...add other font families if needed... */

      /* To override entire default namespaces (e.g., all default spacing), use: */
      /* --spacing-*: initial; */
    }
    ```
    The names of the CSS variables in the `@theme` block determine the generated utility classes (e.g., `--color-foo` makes `bg-foo`, `text-foo` available).

## Pitfalls & Troubleshooting (Lessons Learned)

During the setup with Tailwind CSS v4.1.6 (alpha), several issues were encountered:

1.  **Legacy Directives:** The presence of old Tailwind v3 directives (`@tailwind base;`, `@tailwind components;`, `@tailwind utilities;`) in `input.css` alongside `@import "tailwindcss";` will cause build failures or incorrect output. **Ensure they are completely removed.**
2.  **`tailwind.config.js` Conflicts:** While v4 *can* load a JS config via `@config` in `input.css` for backward compatibility, it's not the primary method and can lead to conflicts, especially with v3-style plugins. For a clean v4 setup, it's best to:
    *   Migrate theme configurations to the `@theme` block in `input.css`.
    *   Remove or rename `tailwind.config.js` to avoid it being inadvertently processed or causing confusion.
3.  **CSS Parsing Sensitivity (v4 Alpha):**
    *   Initially, there were build errors (`Unterminated string: "; */"`) that seemed related to comments or complex string values within `input.css`.
    *   **Recommendation:** Keep the `@theme` block clean and comment-free initially if experiencing issues. Ensure strings (like font lists) are correctly quoted. The issue was ultimately the legacy directives, but keeping the CSS clean is good practice with alpha software.
4.  **Incorrect/Minimal CSS Output:** If `tailwind.min.css` is extremely small (e.g., ~2 lines or ~9KB) and doesn't contain your custom theme variables or utility classes:
    *   Verify `input.css` for the legacy directives mentioned above.
    *   Ensure the `@theme` block is correctly defined.
    *   Check that `@source` directives are pointing to the correct paths for your templates.
    *   Ensure no old, incorrect `input.css` file exists in unexpected locations (e.g., `static/css/`).
5.  **Browser Cache:** After generating a new `tailwind.min.css`, browsers may aggressively cache the old version.
    *   **Always perform a hard refresh** (Ctrl+Shift+R or Cmd+Shift+R).
    *   Use browser developer tools (Network tab -> "Disable cache") when testing CSS changes.
6.  **Linter Warnings:** CSS linters not yet updated for Tailwind v4 syntax may flag new at-rules like `@source` or `@theme` as unknown. This is usually a linter issue and doesn't mean Tailwind itself won't process them correctly.

## Making Future Style Changes

1.  **To change theme-wide styles (colors, fonts, global defaults):**
    *   Modify the CSS custom property definitions within the `@theme { ... }` block in `selfdrive/chauffeur/concierge/css/input.css`.
    *   Example: To change the primary background color, edit `--color-1: #yourNewHex;`.

2.  **To change the styling of specific HTML elements:**
    *   Apply or change Tailwind utility classes directly in your HTML files (e.g., `selfdrive/chauffeur/concierge/templates/index.html`).
    *   Example: Change `<div class="bg-1 p-4">` to `<div class="bg-2 p-6">`.

3.  **To add custom, reusable CSS components or utilities (not directly part of the theme):**
    *   You can define these directly in `css/input.css` below the `@theme` block, potentially using Tailwind's `@layer components { ... }` or `@layer utilities { ... }` for organization, or the new v4 `@utility` directive for custom utilities that can work with variants. Refer to Tailwind v4 documentation for best practices here.

4.  **After any changes to `css/input.css` or if you've used new utility classes in your HTML that you want to ensure are generated:**
    *   **Always re-run the build command:** `npm run build:css` (from `selfdrive/chauffeur/concierge/`)
    *   Perform a hard refresh in your browser.

By following these guidelines, you should be able to maintain and extend the styling for The Concierge effectively.

## Adding a New Static Page

To add a new static HTML page to The Concierge (e.g., a new settings or information page):

1.  **Create the HTML Template:**
    *   Add a new `.html` file in the `selfdrive/chauffeur/concierge/templates/` directory (e.g., `your_new_page.html`).
    *   Structure your HTML as needed. You can use existing pages like `index.html` or `diagnostics.html` as a starting point for layout and including CSS/JS assets:
        ```html
        <!DOCTYPE html>
        <html lang="en" class="h-full">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <title>Concierge - Your New Page</title>
            <link href="/static/css/tailwind.min.css" rel="stylesheet">
            <script src="/static/js/htmx.min.js" defer></script> <!- Optional ->
            <script src="/static/js/app.js" defer></script>     <!- Optional ->
            <link rel="icon" href="/static/favicon.ico"/>
        </head>
        <body class="h-full bg-1 text-primary font-sans">
            <nav class="bg-1 border-b border px-4 py-2 flex items-center justify-between">
              <h1 class="text-xl font-semibold tracking-tight">The Concierge - Your New Page</h1>
              <a href="/" class="text-sm text-accent hover:text-accent-2">Back to Dashboard</a>
            </nav>
            <main class="container mx-auto py-8 px-4">
              <!-- Your page content here -->
              <p>Content for your new page.</p>
            </main>
        </body>
        </html>
        ```
    *   Use Tailwind CSS utility classes for styling as needed.

2.  **Add a Route in `main.py`:**
    *   Open `selfdrive/chauffeur/concierge/main.py`.
    *   Add a new FastAPI route handler function for your page. This function will load and render your new HTML template.
    *   Example for a page at `/your-new-page`:
        ```python
        @app.get("/your-new-page", response_class=HTMLResponse)
        async def your_new_page_handler(request: Request):
            tmpl = templates.get_template("your_new_page.html")
            return tmpl.render(request=request) # Add any other context variables if needed
        ```
    *   Ensure you import `HTMLResponse` and `Request` from `fastapi` if not already done near the top of `main.py` (they are already imported in the current file).
    *   The `templates` object (Jinja2 Environment) is already globally defined in `main.py`.

3.  **Update Navigation (Optional):**
    *   If you want to link to this new page from the main dashboard or other pages, update the relevant HTML templates (e.g., `index.html`) to include links to `/your-new-page`.

4.  **Build CSS (If Necessary):**
    *   If you used any *new* Tailwind CSS classes in `your_new_page.html` that weren't previously used in other scanned files (`index.html`, `app.js`, etc.), you'll need to rebuild the Tailwind CSS:
        ```bash
        # From selfdrive/chauffeur/concierge/ directory
        npm run build:css
        ```
    *   If you only used existing classes, this step might not be strictly necessary, but it's good practice to rebuild if unsure.

5.  **Test:**
    *   Run the Concierge server (e.g., `python -m selfdrive.chauffeur.concierge.main --dev`).
    *   Navigate to `http://localhost:5055/your-new-page` in your browser.
    *   Perform a hard refresh (Ctrl+Shift+R or Cmd+Shift+R) to ensure you're not seeing cached versions, especially after CSS changes.
